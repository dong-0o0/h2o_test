실제랑 완전히 똑같이 테스트 시작

소프트웨어 개발에서 버전 관리 시스템은 팀 협업의 핵심 도구입니다.
여러 개발자가 동시에 같은 파일을 수정할 때 충돌(conflict)이 발생할 수 있습니다.
Git은 이러한 충돌을 감지하고 개발자에게 알려주는 강력하고 유연한 도구입니다.
GitHub, GitLab 등 원격 저장소와 함께 사용하면 더욱 강력합니다.

충돌이 발생하는 주요 상황:
1. 두 명의 개발자가 같은 줄을 다르게 수정한 경우
2. 한 명은 파일을 삭제하고, 다른 한 명은 수정한 경우
3. 브랜치 병합(merge) 또는 리베이스(rebase) 중 변경 사항이 겹치는 경우

충돌 해결 방법:
- 충돌이 발생한 파일을 열어 충돌 마커(<<<<<<<, =======, >>>>>>>)를 확인합니다.
- 어떤 변경 사항을 유지할지 결정합니다.
- 충돌 마커를 제거하고 최종 내용을 저장합니다.
- git add 명령으로 해결된 파일을 스테이징합니다.
- git commit으로 병합을 완료합니다.
